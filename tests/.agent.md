# Agent Guidelines for tests/

## Role of the Coder

When operating in this directory, the `coder` agent is responsible for creating and executing simple, script-based tests to validate the functionality of the refactored code.

### Test Philosophy
- **Validation**: Validation is performed by comparing the output files generated by a test run against known-good benchmark files.

### Workflow
1.  **Use Module Entry Point**: Run the module under test via its `if __name__ == "__main__"` entry point (e.g., `python -m analysis.peak_heights <input>` or `python src/analysis/peak_heights.py <input>`), passing the appropriate sample argument.
2.  **Generate Outputs Manually**: Let the module produce its outputs in the configured workspace (no pytest harness required).
3.  **Inspect & Compare**: Manually inspect the generated outputs against benchmarks/fixtures (e.g., files in `tests/test_analysis/`) using diff tools or direct inspection.
4.  **Report Findings**: Note any discrepancies and summarize whether behavior matches the legacy expectation.
