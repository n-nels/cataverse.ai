# Technical Specification for tests/

This document defines the structure and conventions for all tests in this directory.

## 1. Directory Structure

-   **Test Input Data**: All data required to run tests are in `tests/analysis/`. E.g., *.wxyz where wxyz is four digit integer
-   **Test Input Metadata**: All metadata required to run tests are in 'tests/analysis/'. E.g., *.txt
-   **Benchmark Outputs**: All known-good output files for comparison must are in `tests/analysis/`. E.g., *.csv
