# `src/instrument/` Overview

High-level guide to the instrument modules. This file is intentionally brief and
focuses on module responsibilities rather than refactor history.

## Module Responsibilities

- `main.py`: Bootstraps the OPUS server and starts the ZMQ loop.
- `server.py`: Handles message routing, request/response framing, and the
  top-level server loop.
- `acquisition.py`: Executes background/sample measurements and ties together
  acquisition steps (save, subtract, metadata extraction).
- `client.py`: Low-level OPUS pipe command wrappers and response parsing.
- `dispatch.py`: Manages analysis queues and triggers asynchronous analysis work.
- `state.py`: Holds runtime state for the server (paths, queues, counters).
- `paths.py`: Builds path and configuration containers used by the server.

## Notes

- Configuration is driven by YAML in `config/` via `src.core.config`.
- Output filenames and OPUS command semantics remain compatible with legacy
  workflows.
