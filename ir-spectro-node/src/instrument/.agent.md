# Agent Guidelines for src/instrument

This file directs the behavior of AI agents operating within the `src/instrument/`
directory.

## Validation Protocol

For all operational validation of modules in this directory, the `@validator`
subagent must be used to confirm behavior parity and runtime safety.

### Core Mandates:

1. **Preserve Instrument Logic**: Do not change the logic for functions that rely
   on OPUS pipe commands. Command strings and side effects must remain unchanged.
2. **Reference the Specification**: Before making any changes, read and follow
   the requirements in `.spec.md`. The spec is the ground truth for behavior.
3. **Output Stability**: Output files, schemas, and filenames must remain
   consistent. Changes are considered breaking and require explicit approval.
4. **No Runtime Execution**: Under no circumstances should `opus_wrapper.py` be
   executed in this environment. All changes must be reviewed statically.

## Operational Protocol:

- **On Task Start**: Read `.spec.md` to understand module requirements.
- **Before Modifying**: Identify validation steps and parity constraints.
- **After Modifying**: Invoke `@validator` and `@reviewer` before proceeding.
