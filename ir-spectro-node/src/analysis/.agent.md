# Agent Guidelines for src/analysis

This file directs the behavior of AI agents operating within the `src/analysis/` directory.

## Validation Protocol

For all scientific and numerical validation of modules in this directory, the `@validator` subagent must be used.

### Core Mandates:

1.  **Preserve Scientific Integrity**: All code changes must be validated against established scientific principles. The underlying models (e.g., Voigt, Gaussian) are not to be altered without explicit, high-level approval.
2.  **Reference the Specification**: Before making any changes, you **must** read and understand the technical and scientific requirements outlined in `.spec.md`. The spec is the ground truth for this module's behavior.
3.  **Data-Driven Validation**: Any modification to fitting algorithms, baseline corrections, or data processing pipelines must be validated using representative benchmark data. Changes that cause deviations in results must be justified and documented.
4.  **No "Magic Numbers"**: Do not introduce unexplained constants. All numerical values should be derived from the input data, physical constants, or be clearly defined in the `.spec.md` or a configuration file.
5.  **Output Stability**: The format and structure of the output files (e.g., fitted parameters, plots) must remain consistent. Changes to the output format are considered breaking changes and must be approved.

## Operational Protocol:

-   **On Task Start**: Read `.spec.md` to understand the module's requirements.
-   **Before Modifying**: Identify the relevant validation steps for the module being changed.
-   **After Modifying**: Run the available analysis validation steps and compare the output to the established benchmarks. Manual smoke tests via the module's `if __name__ == "__main__"` entry point are encouraged (or performed by the user) to validate CLI/runtime behavior.
