# Technical Specification for `src/analysis/`

This document provides the technical and scientific specifications for all modules within the `src/analysis/` directory. It serves as the ground truth for agents and developers.

## 1. Spectral Fitting (legacy module: `voight_fit.py`)

### 1.1. Core Scope

- **Purpose**: Perform automated fitting of spectral data, primarily for identifying and quantifying carbonyl (`C=O`) stretching modes and integrated values.
- **Primary Entry Point**: `voight_fit()` (legacy module name retained for compatibility; orchestration lives in `main.py`).
- **Core Model**: Fit uses a **Voigt profile** (Gaussian ⊗ Lorentzian).
  - `V(x; A, μ, σ, γ) = A * (G(x; μ, σ) ⊗ L(x; γ))`
  - **Naming clarity**: Keep the legacy filename `voight_fit.py`, but refer to the analysis as spectral fitting/Voigt profile in documentation and identifiers.

### 1.2. Input and Output Constraints (Keep Stable)

- **Input**: Two-column spectrum data (wavenumber, absorbance); ROI must remain 1750–2250 cm⁻¹ for subIFG/FSD.
- **Output**: Preserve all four `*_Carbonyl*.csv` outputs (schemas, filenames, sort order).
- **Baseline**: `pybaselines.classification.std_distribution` with legacy defaults (config-driven).
- **Isotopes**: 12CO base peaks shifted to 13CO using `analysis.yaml` rules; maintain base-isotope conventions and upper-inclusive rule flag behavior.

**Status**: Phase 5.2.D documentation complete. Next optional subphases: 5.2.E (API separation) and 5.2.F (professionalization).

### 1.3. Phase 5.2.D (Module Decomposition + Orchestration Skeleton) — Complete

**Objective**: Decompose `voight_fit_v3.py` into focused modules plus an orchestrator that preserves behavior while making analysis stages explicit.

**Module Map**: `io.py`, `spectral_fitting.py`, `kinetics_fitting.py`, `output.py`, `main.py`.

**Orchestrator**: `DataAnalysisRunner` in `main.py`.
- **Stages**: spectral deconvolution → kinetics fit → spectral peak heights.
- **Methods**:
  - `run_spectral_fit(file_path)` → returns cumulative peak-area data.
  - `run_kinetics_fit(cumulative_peak_area)` → runs `kinetics_fitting.pfo_fit`.
  - `run_spectral_peak_heights(file_path)` → runs `peak_heights.PeakHeightsAnalyzer().run(...)`.
  - `run_main(file_path)` → enforced order: spectral fit → kinetics fit → peak heights.

**Constraint**: Orchestrator wires existing modules; do not move or change algorithm logic.
